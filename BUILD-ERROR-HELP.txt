If Vercel still says: Build Failed — npm run build exited with 1
================================================================

1) Open the failed deployment on vercel.com → tab "Build Logs".

2) Scroll up and find the FIRST red error line (not the last line).
   Common messages:
   - "vite: not found" / "Cannot find module" → your GitHub copy is missing
     the latest package.json (vite must be inside "dependencies") or you did
     not upload package-lock.json after npm install.
   - "Output Directory does not exist" → Vercel Root Directory is wrong.
   - "Could not read package.json" → wrong folder connected to Vercel.

3) Vercel → Settings → General → Root Directory
   - If GitHub shows package.json NEXT TO the cot-leave folder (wrapper layout):
     Root Directory = empty
   - If GitHub shows ONLY the inside of cot-leave at the top level:
     Root Directory = empty
   - If GitHub top level is a folder that CONTAINS cot-leave:
     Root Directory = either empty (if vercel.json is at that level) OR "cot-leave"

4) Always upload these from the SAME project folder after changes:
   - package.json
   - package-lock.json  (run "npm install" once locally to create/update it)
   - vercel.json
   - src folder, index.html, vite.config.js, public folder

5) Copy the FIRST red error line and search it or send it for help.
